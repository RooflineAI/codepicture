# Milestone v1.0: MVP

**Status:** SHIPPED 2026-01-30
**Phases:** 1-6 (including 1.1)

## Overview

codepicture transforms code snippets into polished, presentation-ready images via a single CLI command. The v1.0 MVP delivers the full pipeline: foundation types and configuration, syntax highlighting and theming, layout calculations, rendering with visual effects, CLI orchestration, and a custom MLIR lexer.

## Milestone Summary

**Decimal Phases:**

- Phase 1.1: Testing Infrastructure (inserted after Phase 1 for test-first development)

**Key Decisions:**

- Python over Rust: Performance not critical, better SVG ecosystem
- Cairo/Pango for rendering: Mature, handles fonts well, supports PNG/SVG/PDF
- Pygments for highlighting: Standard Python choice, extensive language support, custom lexers
- Typer for CLI: Modern, type-hint based, good DX
- Pydantic for config: Validation at load time, clear error messages
- Protocol-based architecture: Enables future backends and testability
- Cairo text API over PyGObject Pango: macOS library linking issues
- Pygments RegexLexer for MLIR: 80% coverage with 20% effort vs Sublime parser
- Fixed shadow style (on/off only): Follows design decision, reduces config complexity

**Issues Resolved:**

- macOS Pango library linking: switched to Cairo text API
- Pygments trailing empty lines: trimmed unless code ends with newline
- MLIR float/hex patterns matching as dialect operations: reordered token patterns
- Cairo BGRA pre-multiplied alpha: converted via Pillow RGBa mode

**Issues Deferred:**

- None — all v1 requirements shipped

**Technical Debt Incurred:**

- toml_theme.py at 19% test coverage (TOML file loading paths — supplementary feature)
- loader.py at 56% coverage (_get_base_themes helper untested)
- PangoTextMeasurer named for Pango but uses Cairo text API (macOS workaround)
- Shadow is on/off only with fixed macOS style (not user-configurable)
