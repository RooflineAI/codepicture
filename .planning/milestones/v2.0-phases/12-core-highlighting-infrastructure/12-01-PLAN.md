<feature>
  <name>Line Range Parser and Highlight Color Resolver</name>
  <files>src/codepicture/render/highlights.py, tests/test_highlights.py</files>
  <behavior>
    parse_line_ranges(specs, total_lines, line_number_offset=1) -> set[int]:
      - ["3"] with total_lines=10 -> {2}
      - ["7-12"] with total_lines=15 -> {6, 7, 8, 9, 10, 11}
      - ["3", "7-12", "15"] with total_lines=15 -> {2, 6, 7, 8, 9, 10, 11, 14}
      - ["1"] with total_lines=5, line_number_offset=0 -> {1}
      - ["0"] with total_lines=5, line_number_offset=0 -> {0}
      - [] -> set() (empty input returns empty set)
      - ["0"] with total_lines=5, line_number_offset=1 -> InputError (out of bounds)
      - ["11"] with total_lines=10 -> InputError (out of bounds)
      - ["5-3"] with total_lines=10 -> InputError (reversed range)
      - ["abc"] with total_lines=10 -> InputError (invalid syntax)
      - [" 3 "] with total_lines=10 -> {2} (whitespace stripped)

    resolve_highlight_color(color_str) -> Color:
      - None -> DEFAULT_HIGHLIGHT_COLOR (Color(r=255, g=230, b=80, a=64))
      - "#FF000040" -> Color(r=255, g=0, b=0, a=64)
      - "#FF0000" -> Color(r=255, g=0, b=0, a=64) (6-char hex gets default alpha)
      - "#00FF00" -> Color(r=0, g=255, b=0, a=64) (6-char hex gets default alpha)

    DEFAULT_HIGHLIGHT_COLOR = Color(r=255, g=230, b=80, a=64)  # #FFE65040 -- warm yellow at ~25% opacity
  </behavior>
  <implementation>
    Create src/codepicture/render/highlights.py with:

    1. DEFAULT_HIGHLIGHT_COLOR constant: Color(r=255, g=230, b=80, a=64)
       - This is #FFE650 at ~25% alpha (64/255), a warm yellow per CONTEXT.md

    2. HIGHLIGHT_CORNER_RADIUS constant: 0
       - Sharp rectangles now, but extracted as a constant so Phase 13/14 can change it
       - Not used in this file, but exported for renderer to use

    3. parse_line_ranges(specs: list[str], total_lines: int, line_number_offset: int = 1) -> set[int]:
       - Import re from stdlib, InputError from codepicture.errors
       - For each spec in specs:
         a. Strip whitespace
         b. Validate format with regex r"^\d+(-\d+)?$" -- raise InputError if no match
         c. If contains "-": split on "-" (maxsplit=1), parse start_num and end_num
            - If start_num > end_num: raise InputError with "start must not exceed end"
            - For each line_num in range(start_num, end_num + 1):
              Convert to 0-based: idx = line_num - line_number_offset
              If idx < 0 or idx >= total_lines: raise InputError with valid range info
              Add idx to result set
         d. Else: parse single line_num, convert, validate, add to set
       - Return result set
       - Empty specs list returns empty set (no error)

    4. resolve_highlight_color(color_str: str | None) -> Color:
       - Import Color from codepicture.core.types
       - If color_str is None: return DEFAULT_HIGHLIGHT_COLOR
       - Parse with Color.from_hex(color_str)
       - If user provided 6-char hex (len(color_str.lstrip("#")) == 6):
         Override alpha to 64 (default ~25% opacity)
       - Return the color

    Follow existing codebase patterns:
    - Use __all__ for public API
    - Type hints on all parameters and returns
    - Docstrings matching project style (Google-style with Args/Returns/Raises)
  </implementation>
</feature>
