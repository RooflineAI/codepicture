## v1.1 Requirements

### Bug Fixes & Safety

- [x] **SAFE-01**: Fix MLIR lexer hang on `test.mlir` — diagnosed as uncached font resolution; fixed with lru_cache + canvas font caching + lexer catch-all rule. 186x speedup (40s → 0.22s)
- [x] **SAFE-02**: Application-level rendering timeout guard — ThreadPoolExecutor-based wrapper with configurable timeout (default 30s), clean abort with RenderTimeoutError
- [x] **SAFE-03**: RenderTimeoutError in error hierarchy — subclass of RenderError, user-friendly CLI output with exit code 2, no Python tracebacks
- [x] **SAFE-04**: pytest-timeout with 5s default — kills hanging tests, 14 slow tests marked with 15s override
- [x] **SAFE-05**: CI job timeout (10 minutes) — GitHub Actions workflow with artifact upload on failure

### Visual Regression Testing

- [x] **VRT-01**: Visual regression infrastructure — pixelmatch-based comparison with threshold=0.1, fail_percent=0.001, composite diff output
- [x] **VRT-02**: Python reference images — 3 default refs (PNG/SVG/PDF) generated and tested
- [x] **VRT-03**: Rust reference images — 3 default refs generated and tested
- [x] **VRT-04**: C++ reference images — 3 default refs generated and tested
- [x] **VRT-05**: JavaScript reference images — 3 default refs generated and tested
- [x] **VRT-06**: MLIR reference images — 3 default refs generated and tested
- [x] **VRT-07**: Config variant references — 5 variant refs (shadow-off, chrome-off, lines-off, combos) generated and tested
- [x] **VRT-08**: Snapshot update mechanism — `--snapshot-update` pytest flag regenerates all reference images

### Rendering Reliability

- [x] **REL-01**: 5 languages x 3 formats parametrized tests — 15 format validation tests + 15 timeout tests in reliability matrix
- [x] **REL-02**: Feature toggle parametrized tests — 7 toggle combo tests + dimension change assertions
- [x] **REL-03**: Error handling audit — file not found (InputError, exit 1), unknown language (fallback + stderr warning), timeout (RenderTimeoutError, exit 2), all clean messages

### Performance

- [x] **PERF-01**: Per-stage benchmarks — 18 benchmarks (5 highlight + 5 layout + 5 render + 3 feature toggles) with pre-computed input isolation
- [x] **PERF-02**: End-to-end pipeline benchmarks — 3 benchmarks (small 12 lines, medium 60 lines, large 267 lines)
- [x] **PERF-03**: Benchmark CI integration — standalone workflow (manual + weekly cron), JSON + text artifacts, 90-day retention
