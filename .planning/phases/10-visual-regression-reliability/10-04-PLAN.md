  <action>
    Create `.gitattributes` with:
    ```
    # Visual regression reference images tracked via Git LFS
    tests/visual/references/*.png filter=lfs diff=lfs merge=lfs -text
    ```

    Then initialize LFS tracking:
    ```bash
    git lfs install
    git lfs track "tests/visual/references/*.png"
    ```

    Note: If git-lfs is not installed, skip LFS setup and just create the .gitattributes file. The reference images are small enough (~20 PNGs, each ~50-200KB) that they can live in regular Git if LFS isn't available. Add a comment in .gitattributes noting LFS is recommended.

<task type="auto">
  <name>Task 2: Add visual regression CI job to GitHub Actions workflow</name>
  <files>
    .github/workflows/tests.yml
  </files>
  <action>
    Update `.github/workflows/tests.yml` to add a `visual` job that runs in parallel with the existing `test` job:

    ```yaml
    visual:
      runs-on: ubuntu-latest
      timeout-minutes: 10

      steps:
        - uses: actions/checkout@v4
          with:
            lfs: true

        - name: Install uv
          uses: astral-sh/setup-uv@v5
          with:
            enable-cache: true

        - name: Set up Python
          run: uv python install 3.13

        - name: Install system dependencies
          run: |
            sudo apt-get update
            sudo apt-get install -y libcairo2-dev libpango1.0-dev fonts-jetbrains-mono

        - name: Install dependencies
          run: uv sync --locked --dev

        - name: Run visual regression tests
          run: uv run pytest tests/visual/ -v --tb=short 2>&1 | tee visual-test-output.txt
          env:
            # Ensure consistent font rendering
            PANGOCAIRO_BACKEND: fontconfig

        - name: Upload diff composites on failure
          if: failure()
          uses: actions/upload-artifact@v4
          with:
            name: visual-diff-composites
            path: |
              /tmp/pytest-*/pytest-*/visual_diffs/
              visual-test-output.txt
            retention-days: 7
    ```

    Keep the existing `test` job unchanged. Both jobs run in parallel (no `needs` dependency between them).

    Important: The `fonts-jetbrains-mono` package ensures consistent font rendering in CI. The `PANGOCAIRO_BACKEND=fontconfig` env var ensures Pango uses fontconfig for font resolution.
  </action>
  <verify>
    - `.github/workflows/tests.yml` contains both `test` and `visual` jobs
    - The `visual` job installs system deps including fonts-jetbrains-mono
    - The `visual` job uploads diff artifacts on failure
    - Existing `test` job is unchanged
  </verify>
  <done>CI workflow has a parallel visual regression job that installs fonts, runs visual tests, and uploads diff composites on failure. Visual test failures block PR merge.</done>
