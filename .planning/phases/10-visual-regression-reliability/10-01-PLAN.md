  <action>
    1. Add `--snapshot-update` option to `tests/conftest.py` via `pytest_addoption`:
    ```python
    def pytest_addoption(parser):
        parser.addoption("--snapshot-update", action="store_true", default=False,
                         help="Update visual regression reference images")
    ```

    2. Create `tests/visual/conftest.py` with:

    **Fixtures:**
    - `snapshot_update(request)` — reads `--snapshot-update` flag
    - `visual_fixtures_dir()` — returns Path to `tests/visual/fixtures/`
    - `references_dir()` — returns Path to `tests/visual/references/`
    - `diff_output_dir(tmp_path)` — returns `tmp_path / "visual_diffs"` for failure artifacts

    **Helper functions (module-level, not fixtures):**

    - `compare_images(actual_image: Image, reference_path: Path, diff_output_dir: Path, test_name: str, threshold: float = 0.1, fail_percent: float = 0.001) -> tuple[bool, float]`:
      - Opens reference_path as RGBA
      - Converts actual_image to RGBA
      - If sizes differ, returns (False, 100.0) and saves a text note about size mismatch
      - Calls `pixelmatch(reference, actual, diff_img, threshold=threshold, includeAA=False)`
      - Computes mismatch_pct = (num_diff / total_pixels) * 100
      - Always prints mismatch_pct (even on pass) for drift monitoring
      - If failed, builds a side-by-side composite (expected | actual | diff) and saves to diff_output_dir / f"{test_name}_composite.png"
      - Returns (passed, mismatch_pct)

    - `build_composite(expected: Image, actual: Image, diff: Image) -> Image`:
      - Creates new image 3x width, concatenates expected | actual | diff horizontally
      - Adds "Expected", "Actual", "Diff" labels at top of each section (use Pillow ImageDraw)

    - `svg_to_png(svg_data: bytes, scale: float = 2.0) -> Image`:
      - Uses `cairosvg.svg2png(bytestring=svg_data, scale=scale)`
      - Returns `Image.open(BytesIO(png_bytes)).convert("RGBA")`

    - `pdf_to_png(pdf_data: bytes, dpi: int = 144) -> Image`:
      - Uses `pymupdf.open(stream=pdf_data, filetype="pdf")`
      - Gets page 0, renders with matrix for target dpi
      - Returns as RGBA Pillow Image

    - `render_fixture(fixture_path: Path, config: RenderConfig, language: str | None = None) -> tuple[bytes, str]`:
      - Reads fixture_path, runs the full pipeline (register fonts, highlight, layout, render) and returns (data_bytes, format_extension)
      - This avoids writing to disk — returns raw render bytes for comparison

    **Important:** The `threshold` parameter in pixelmatch controls perceptual color distance (0.1 is the library default, good starting point). The `fail_percent` controls what percentage of differing pixels is acceptable (0.001 = 0.001% of total pixels).
