  <action>
    Create PygmentsHighlighter class implementing the Highlighter protocol:

    1. Define TokenInfo dataclass (frozen, slots) with fields:
       - text: str
       - token_type: Any (Pygments token type)
       - line: int (0-indexed)
       - column: int (0-indexed)

    2. Implement highlight(code: str, language: str) -> list[list[TokenInfo]]:
       - Resolve language alias via resolve_language_alias()
       - Get lexer using pygments.lexers.get_lexer_by_name()
       - Catch ClassNotFound, raise HighlightError with available languages hint
       - Iterate over lexer.get_tokens(code), tracking line:column positions
       - Split tokens on newlines (per RESEARCH.md Pattern 1 and Pitfall 3)
       - Handle empty lines (per RESEARCH.md Pitfall 4)
       - Return list of lines, each line a list of TokenInfo

    3. Implement detect_language(code: str, filename: str | None = None) -> str:
       - If filename provided, use pygments.lexers.get_lexer_for_filename()
       - If no filename, use pygments.lexers.guess_lexer(code)
       - Return lexer.aliases[0] (first alias is canonical name)
       - Catch ClassNotFound, raise HighlightError

    4. Implement list_languages() -> list[str]:
       - Use pygments.lexers.get_all_lexers() to collect all aliases
       - Return sorted unique list

    5. Update __init__.py to export PygmentsHighlighter and TokenInfo

    6. Update src/codepicture/__init__.py to export from highlight module

    7. Update src/codepicture/core/protocols.py Highlighter protocol:
       - Add import for TokenInfo (use TYPE_CHECKING guard to avoid circular import):
         ```python
         if TYPE_CHECKING:
             from codepicture.highlight.pygments_highlighter import TokenInfo
         ```
       - Change highlight() return type from `list[list[tuple[str, Any]]]` to `list[list["TokenInfo"]]`
       - Update docstring: "Returns: List of lines, where each line is a list of TokenInfo objects containing text, token_type, line, and column."

    Reference RESEARCH.md Pattern 1 (Token Position Tracking) for implementation details.
