  <action>
Create end-to-end pipeline benchmarks that call `generate_image` directly (not `generate_image_with_timeout` -- no need for timeout in benchmarks).

The benchmarks write to a temp file, measuring the full pipeline: font registration, highlighting, layout, rendering, and file write.

```python
import pytest
import tempfile
from pathlib import Path
from codepicture import RenderConfig, register_bundled_fonts
from codepicture.cli.orchestrator import generate_image

SIZES = ["small", "medium", "large"]

@pytest.mark.timeout(0)
@pytest.mark.parametrize("size", SIZES)
def test_bench_pipeline_e2e(benchmark, sized_inputs, size, tmp_path):
    """Benchmark full pipeline for different input sizes."""
    code = sized_inputs[size]
    config = RenderConfig()
    output_file = tmp_path / f"bench_{size}.png"

    def run():
        generate_image(
            code=code,
            output_path=output_file,
            config=config,
            language="python",
            filename=f"bench_{size}.py",
        )

    benchmark(run)
    # Verify output was actually created
    assert output_file.exists()
    assert output_file.stat().st_size > 0
```

Key points:
- `sized_inputs` comes from conftest, provides small/medium/large code strings
- `tmp_path` is pytest's built-in temp directory fixture
- Output format is PNG (default) -- benchmarking the most common use case
- Language is always "python" since the sized inputs are Python code
- `@pytest.mark.timeout(0)` disables the 5s timeout
- The assertion after benchmark confirms the pipeline actually produced output (not just ran silently)
- `register_bundled_fonts()` is called inside `generate_image`, so it's included in the benchmark -- this is intentional for e2e measurement (it's cached via lru_cache anyway)

Create a standalone GitHub Actions workflow for benchmarks.

```yaml
name: Benchmarks

on:
  workflow_dispatch:
  schedule:
    - cron: '0 6 * * 1'  # Weekly on Monday at 6am UTC

jobs:
  benchmark:
    runs-on: ubuntu-latest
    timeout-minutes: 15

    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v5
        with:
          enable-cache: true

      - name: Set up Python
        run: uv python install 3.13

      - name: Install system dependencies
        run: |
          sudo apt-get update
          sudo apt-get install -y libcairo2-dev libpango1.0-dev fonts-jetbrains-mono

      - name: Install dependencies
        run: uv sync --locked --dev

      - name: Run benchmarks
        run: |
          uv run pytest tests/benchmarks/ \
            --benchmark-only \
            --benchmark-json=benchmark-results.json \
            -v 2>&1 | tee benchmark-output.txt

      - name: Upload benchmark results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: benchmark-results-${{ github.sha }}
          path: |
            benchmark-results.json
            benchmark-output.txt
          retention-days: 90
```

Key design decisions:
- Separate workflow file (not in test.yml) -- independent from main CI
- `workflow_dispatch` for manual runs + weekly schedule for tracking
- `timeout-minutes: 15` -- generous but bounded
- System deps include Cairo/Pango/fonts (same as visual regression job)
- `--benchmark-json` exports machine-readable results
- Console output also captured in text file for quick review
- `if: always()` on upload so partial results are captured even on failure
- 90-day retention for artifacts -- enough to track trends
- No regression gating -- purely informational per CONTEXT.md decisions
