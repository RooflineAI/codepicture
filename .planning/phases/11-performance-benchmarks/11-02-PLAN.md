**test_bench_highlight.py:**
Create parametrized benchmark tests for the highlight stage.

```python
import pytest

LANGUAGES = [
    ("python", "python"),
    ("rust", "rust"),
    ("cpp", "cpp"),
    ("javascript", "javascript"),
    ("mlir", "mlir"),
]

@pytest.mark.timeout(0)
@pytest.mark.parametrize("lang_key,lang_id", LANGUAGES, ids=[l[0] for l in LANGUAGES])
def test_bench_highlight(benchmark, highlighter, code_samples, lang_key, lang_id):
    code = code_samples[lang_key]
    benchmark(highlighter.highlight, code, lang_id)
```

Key points:
- Use `@pytest.mark.timeout(0)` on each test function to disable the 5s pytest-timeout
- Use `benchmark` fixture (provided by pytest-benchmark) as first arg
- `highlighter` and `code_samples` come from conftest
- The `benchmark()` call wraps the function under test -- pytest-benchmark handles iteration/warmup
- Language keys must match the conftest `code_samples` dict keys

```python
import pytest
from codepicture import Renderer, RenderConfig, get_theme

LANGUAGES = ["python", "rust", "cpp", "javascript", "mlir"]

@pytest.mark.timeout(0)
@pytest.mark.parametrize("lang", LANGUAGES)
def test_bench_render(benchmark, pre_tokenized, pre_computed_metrics, default_config):
    tokens = pre_tokenized[lang]
    metrics = pre_computed_metrics[lang]
    theme = get_theme(default_config.theme)
    renderer = Renderer(default_config)
    benchmark(renderer.render, tokens, metrics, theme)
```

**Set 2: Feature toggle combinations (Python only)**
Three configs:
- `minimal`: shadow=False, show_chrome=False, show_line_numbers=False
- `with_line_numbers`: shadow=False, show_chrome=False, show_line_numbers=True
- `full`: shadow=True, show_chrome=True, show_line_numbers=True (default)

```python
FEATURE_CONFIGS = [
    ("minimal", {"shadow": False, "show_chrome": False, "show_line_numbers": False}),
    ("with_line_numbers", {"shadow": False, "show_chrome": False, "show_line_numbers": True}),
    ("full", {"shadow": True, "show_chrome": True, "show_line_numbers": True}),
]

@pytest.mark.timeout(0)
@pytest.mark.parametrize("config_name,overrides", FEATURE_CONFIGS, ids=[c[0] for c in FEATURE_CONFIGS])
def test_bench_render_features(benchmark, highlighter, code_samples, config_name, overrides):
    config = RenderConfig(**overrides)
    code = code_samples["python"]
    tokens = highlighter.highlight(code, "python")
    measurer = PangoTextMeasurer()
    engine = LayoutEngine(measurer, config)
    metrics = engine.calculate_metrics(tokens)
    theme = get_theme(config.theme)
    renderer = Renderer(config)
    benchmark(renderer.render, tokens, metrics, theme)
```

For the feature toggle tests, we re-compute tokens/metrics with the specific config since layout depends on config (line numbers affect metrics). This is intentional -- setup cost is minimal and we need correct metrics for each config.
