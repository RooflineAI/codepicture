  <action>
1. Add `timeout-minutes: 10` to the `test` job (at the job level, same indentation as `runs-on`).

2. Modify the "Run tests with coverage" step to capture output to a file while still showing it in the console. Replace the `run` command with:
   ```yaml
   run: |
     uv run pytest --cov --cov-report=xml --cov-report=term 2>&1 | tee test-output.txt
     exit ${PIPESTATUS[0]}
   ```
   This pipes output through `tee` so it appears in the console AND is saved to file. `PIPESTATUS[0]` preserves pytest's exit code.

3. Add a new step after the test step and BEFORE the Codecov upload step:
   ```yaml
   - name: Upload test output on failure
     if: failure()
     uses: actions/upload-artifact@v4
     with:
       name: test-output
       path: test-output.txt
       retention-days: 7
   ```
   This only runs when a prior step fails (timeout or test failure).

4. Preserve the existing Codecov upload step unchanged.

The final job structure should be:
- checkout
- install uv
- set up python
- install dependencies
- run tests (with tee)
- upload test output (on failure)
- upload coverage to codecov
