<task type="auto">
  <name>Task 2: Draw highlight rectangles in _render_legacy and _render_wrapped</name>
  <files>src/codepicture/render/renderer.py</files>
  <action>
    In `_render_legacy()`, add highlight drawing as the FIRST operation in the line number loop. BEFORE the existing line number drawing loop, add a dedicated highlight loop (or combine). The cleanest approach: add a highlight drawing block at the very start of _render_legacy, BEFORE the line number drawing code:

    ```python
    # Draw highlight rectangles (behind all text)
    if highlighted_lines:
        for line_idx in range(len(lines)):
            if line_idx in highlighted_lines:
                highlight_y = (
                    metrics.content_y
                    + code_y_offset
                    + line_idx * metrics.line_height_px
                )
                canvas.draw_rectangle(
                    x=metrics.content_x,
                    y=highlight_y,
                    width=metrics.content_width,
                    height=metrics.line_height_px,
                    color=highlight_color,
                    corner_radius=HIGHLIGHT_CORNER_RADIUS,
                )
    ```

    Place this block at the very start of `_render_legacy`, before the `if config.show_line_numbers:` block. This ensures highlights are drawn BEHIND both line numbers and code tokens (correct z-order).

    In `_render_wrapped()`, add the same pattern at the very start, BEFORE the `char_maps` building code:

    ```python
    # Draw highlight rectangles (behind all text)
    if highlighted_lines:
        for display_idx, dline in enumerate(metrics.display_lines):
            if dline.source_line_idx in highlighted_lines:
                highlight_y = (
                    metrics.content_y
                    + code_y_offset
                    + display_idx * metrics.line_height_px
                )
                canvas.draw_rectangle(
                    x=metrics.content_x,
                    y=highlight_y,
                    width=metrics.content_width,
                    height=metrics.line_height_px,
                    color=highlight_color,
                    corner_radius=HIGHLIGHT_CORNER_RADIUS,
                )
    ```

    Critical details:
    - Y position: `content_y + code_y_offset + line_index * line_height_px` (top of line slot)
    - X position: `content_x` (inside padding, spans gutter + code area)
    - Width: `content_width` (full content area width)
    - Height: `line_height_px` (NOT char_height -- must match line spacing exactly)
    - corner_radius: `HIGHLIGHT_CORNER_RADIUS` (0 for now, extracted constant for Phase 13/14)
    - In _render_wrapped: check `dline.source_line_idx in highlighted_lines` -- this automatically highlights ALL display lines for a wrapped source line
    - Drawing BEFORE all text ensures correct z-order (highlights behind text)
    - The highlight color has alpha (e.g. a=64 for ~25% opacity), so it blends with the background through Cairo's default OVER operator
  </action>
