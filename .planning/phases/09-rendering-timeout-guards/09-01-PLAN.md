<task type="auto">
  <name>Task 2: Add timeout wrapper and atomic writes to orchestrator</name>
  <files>src/codepicture/cli/orchestrator.py</files>
  <action>
Modify cli/orchestrator.py to add:

1. **Atomic file write helper** `_write_output_atomic(data: bytes, output_path: Path)`:
   - Create parent dirs with `output_path.parent.mkdir(parents=True, exist_ok=True)`
   - Use `tempfile.NamedTemporaryFile(dir=output_path.parent, prefix=".codepicture-", suffix=output_path.suffix, delete=False)`
   - Write data to temp file, close it, then `shutil.move(tmp_path, str(output_path))`
   - On any exception: close temp file, `os.unlink(tmp_path)`, re-raise
   - Import `os`, `tempfile`, `shutil` at top

2. **Refactor generate_image** to use `_write_output_atomic`:
   - Replace `output_path.parent.mkdir(...)` and `output_path.write_bytes(result.data)` at the end with `_write_output_atomic(result.data, output_path)`

3. **Add generate_image_with_timeout function**:
   - Signature: `generate_image_with_timeout(code, output_path, config, language=None, filename=None, timeout=30.0)`
   - If `timeout is None` (disabled via --timeout 0): call `generate_image()` directly, return
   - Otherwise: create `ThreadPoolExecutor(max_workers=1)` explicitly (NOT with `with` statement — see Pitfall 1 in RESEARCH.md)
   - Submit `generate_image` to executor
   - Call `future.result(timeout=timeout)`
   - On success: call `executor.shutdown(wait=True)`
   - On `concurrent.futures.TimeoutError`: call `executor.shutdown(wait=False, cancel_futures=True)`, then raise `RenderTimeoutError` with message including timeout value, filename, "No output file written.", and suggestion to increase timeout with `--timeout {int(timeout * 2)}`
   - Import `concurrent.futures` at top
   - Import `RenderTimeoutError` from `codepicture.errors`

Keep the existing `generate_image` function unchanged in its core logic — it stays as the inner pipeline. `generate_image_with_timeout` wraps it.
  </action>

  </verify>
  <done>generate_image_with_timeout wraps the pipeline with ThreadPoolExecutor timeout. _write_output_atomic provides temp-file-based atomic writes. Existing generate_image function unchanged in behavior. All existing tests pass.</done>
