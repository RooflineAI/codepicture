**Fix 1 — Cache resolve_font_family() (PRIMARY FIX):**

In `src/codepicture/fonts/__init__.py`:
1. Add `import functools` at the top
2. Add `@functools.lru_cache(maxsize=16)` decorator to `resolve_font_family()`
3. No other changes needed — the decorator caches the entire function body including the `list_fonts()` call

The function signature is `resolve_font_family(requested: str, default: str = "JetBrains Mono") -> str`. Both args are strings, so they are hashable and work with lru_cache.

**Fix 2 — Add catch-all identifier rule to MLIR lexer:**

In `src/codepicture/highlight/mlir_lexer.py`:
1. Add a catch-all identifier rule AFTER the affine expression operators rule (line ~71, after `Operator.Word`) and BEFORE the string rule:
   ```python
   # General identifiers (catch-all for bare words)
   (r"[a-zA-Z_][\w]*", Name),
   ```
2. Also add a rule for the MLIR `...` ellipsis syntax (used as variadic argument marker in test.mlir):
   ```python
   (r"\.\.\.", Punctuation),
   ```
   Place this BEFORE the catch-all identifier rule (after the affine operators).

The catch-all MUST come after all keyword/type/builtin rules so specific patterns match first. Pygments tries rules in order; this only triggers for words that don't match any prior rule.
