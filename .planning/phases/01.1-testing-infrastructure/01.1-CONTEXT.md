<decisions>
## Implementation Decisions

### Test Organization
- Tests live in `tests/` at project root
- Test file naming: `test_<module>.py` (standard pytest convention)
- Shared fixtures in `tests/conftest.py`
- Directory structure mirrors src: `tests/codepicture/test_config.py` matching `src/codepicture/config.py`

### Coverage Thresholds
- Minimum coverage: 80% — builds fail below this
- Coverage regression: fail builds if coverage drops from previous run
- Exclusions: tests/, type stubs, `__main__.py`, and `TYPE_CHECKING` blocks
- Reporting: terminal summary + HTML report for local browsing

### Plugin Selection
- pytest-cov — coverage reporting integration
- pytest-randomly — randomize test order to catch hidden dependencies
- No pytest-xdist (parallel execution not needed yet)
- No pytest-timeout (not needed for this codebase size)

### Pytest Configuration
- Configuration lives in `pyproject.toml` (consolidate all tool config)
- Default options: `-v --tb=short` (verbose output, short tracebacks)
- Define `@pytest.mark.slow` marker for long-running tests (skip with `-m 'not slow'`)

### Claude's Discretion
- Exact fixture implementations for core types
- How to structure tests for protocol compliance
- CI workflow file naming and job organization

</decisions>
