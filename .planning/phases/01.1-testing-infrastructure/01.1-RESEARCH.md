# Phase 1.1: Testing Infrastructure - Research

**Researched:** 2026-01-28
**Domain:** Python testing infrastructure (pytest, coverage, CI/CD)
**Confidence:** HIGH

## Summary

This research covers setting up pytest testing infrastructure for the existing codepicture foundation code. The domain is well-established with excellent tooling and documentation. All recommended tools (pytest, pytest-cov, pytest-randomly) have mature, stable APIs and extensive documentation.

The codebase to test includes:
- **Core types:** `Color` (frozen dataclass with slots), `Dimensions`, `Position`, `Rect`, `TextStyle`, enums (`OutputFormat`, `WindowStyle`)
- **Protocols:** `TextMeasurer`, `Canvas`, `Highlighter`, `Theme` (structural subtyping, not `@runtime_checkable`)
- **Config:** `RenderConfig` (Pydantic v2 model), `load_config()` (TOML loading with precedence)
- **Errors:** Custom exception hierarchy (`CodepictureError`, `ConfigError`, `ThemeError`, `RenderError`, `HighlightError`)
- **Text utilities:** `normalize_tabs()`

**Primary recommendation:** Use pytest with pytest-cov and pytest-randomly, configured entirely in pyproject.toml. Use uv for CI dependency management since the project already uses uv (evidenced by uv.lock). Leverage pytest's built-in `tmp_path` fixture for config file testing.

## Standard Stack

The established libraries/tools for this domain:

### Core

| Library | Version | Purpose | Why Standard |
|---------|---------|---------|--------------|
| pytest | >=8.0 | Test framework | De facto Python standard, excellent fixtures, clear assertions |
| pytest-cov | >=6.0 | Coverage integration | Official coverage.py integration for pytest |
| pytest-randomly | >=3.16 | Test order randomization | Catches hidden test dependencies |

|---------|---------|---------|-------------|

|------------|-----------|----------|

### Pattern 1: Frozen Dataclass Testing

**What:** Testing immutable dataclasses with slots
**When to use:** Testing `Color`, `TextStyle`

```python
# Source: pytest documentation + Python docs
import pytest
from codepicture.core.types import Color

class TestColor:
    """Tests for Color frozen dataclass."""

    def test_from_hex_six_char(self):
        """#RRGGBB format creates correct color."""
        color = Color.from_hex("#ff8800")
        assert color.r == 255
        assert color.g == 136
        assert color.b == 0
        assert color.a == 255  # default alpha

    def test_from_hex_with_alpha(self):
        """#RRGGBBAA format preserves alpha."""
        color = Color.from_hex("#ff880080")
        assert color.a == 128

    def test_from_hex_short_format(self):
        """#RGB expands correctly."""
        color = Color.from_hex("#f80")
        assert color == Color(r=255, g=136, b=0, a=255)

    def test_immutability(self):
        """Frozen dataclass rejects attribute changes."""
        color = Color(r=255, g=0, b=0)
        with pytest.raises(AttributeError):
            color.r = 128  # FrozenInstanceError is subclass of AttributeError

    def test_to_hex_roundtrip(self):
        """to_hex output can be parsed by from_hex."""
        original = Color(r=100, g=150, b=200, a=128)
        hex_str = original.to_hex()
        restored = Color.from_hex(hex_str)
        assert original == restored
```

### Pattern 2: Pydantic Model Validation Testing

**What:** Testing Pydantic model validation behavior
**When to use:** Testing `RenderConfig`

```python
# Source: Pydantic documentation + pytest patterns
import pytest
from pydantic import ValidationError
from codepicture.config.schema import RenderConfig

class TestRenderConfig:
    """Tests for RenderConfig validation."""

    def test_defaults(self):
        """Default config has expected values."""
        config = RenderConfig()
        assert config.theme == "catppuccin-mocha"
        assert config.font_size == 14
        assert config.padding == 40

    def test_font_size_range(self):
        """font_size must be between 6 and 72."""
        with pytest.raises(ValidationError) as exc_info:
            RenderConfig(font_size=5)
        errors = exc_info.value.errors()
        assert any("font_size" in str(e["loc"]) for e in errors)

    def test_extra_fields_rejected(self):
        """Unknown fields cause validation error."""
        with pytest.raises(ValidationError) as exc_info:
            RenderConfig(unknown_field="value")
        errors = exc_info.value.errors()
        assert any(e["type"] == "extra_forbidden" for e in errors)

    def test_output_format_string_conversion(self):
        """String output_format converts to enum."""
        config = RenderConfig(output_format="svg")
        assert config.output_format.value == "svg"
```

### Pattern 3: Config Loading with tmp_path

**What:** Testing config file loading with temporary files
**When to use:** Testing `load_config()`

```python
# Source: pytest docs (tmp_path fixture)
import pytest
from pathlib import Path
from codepicture.config import load_config
from codepicture.errors import ConfigError

class TestLoadConfig:
    """Tests for config loading and merging."""

    def test_load_from_toml(self, tmp_path: Path):
        """Valid TOML file loads successfully."""
        config_file = tmp_path / "config.toml"
        config_file.write_text('''
theme = "dracula"
font_size = 16
''')
        config = load_config(
            global_path=None,
            local_path=config_file,
            use_defaults=False,
        )
        assert config.theme == "dracula"
        assert config.font_size == 16

    def test_invalid_toml_raises_config_error(self, tmp_path: Path):
        """Malformed TOML raises ConfigError."""
        config_file = tmp_path / "bad.toml"
        config_file.write_text("invalid [ toml")
        with pytest.raises(ConfigError) as exc_info:
            load_config(
                global_path=None,
                local_path=config_file,
                use_defaults=False,
            )
        assert "Invalid TOML" in str(exc_info.value)

    def test_cli_overrides_file(self, tmp_path: Path):
        """CLI overrides take precedence over file values."""
        config_file = tmp_path / "config.toml"
        config_file.write_text('font_size = 16')
        config = load_config(
            global_path=None,
            local_path=config_file,
            cli_overrides={"font_size": 20},
            use_defaults=False,
        )
        assert config.font_size == 20  # CLI wins
```

### Pattern 4: Protocol Compliance Testing

**What:** Testing that mock implementations satisfy Protocol contracts
**When to use:** Testing `Canvas`, `TextMeasurer`, `Highlighter`, `Theme` protocols

```python
# Source: PEP 544, typing documentation
from codepicture.core.protocols import TextMeasurer, Theme
from codepicture.core.types import Color, TextStyle

class MockTextMeasurer:
    """Minimal implementation satisfying TextMeasurer protocol."""

    def measure_text(
        self, text: str, font_family: str, font_size: int
    ) -> tuple[float, float]:
        # Simple approximation: char_count * font_size * 0.6, font_size * 1.2
        return (len(text) * font_size * 0.6, font_size * 1.2)


def test_mock_text_measurer_satisfies_protocol():
    """MockTextMeasurer structurally matches TextMeasurer protocol."""
    measurer: TextMeasurer = MockTextMeasurer()
    width, height = measurer.measure_text("hello", "Mono", 14)
    assert isinstance(width, float)
    assert isinstance(height, float)
```

### Pattern 5: Exception Hierarchy Testing

**What:** Testing custom exception inheritance and attributes
**When to use:** Testing `ConfigError`, `ThemeError`, etc.

```python
# Source: pytest patterns
import pytest
from codepicture.errors import (
    CodepictureError,
    ConfigError,
    ThemeError,
    RenderError,
    HighlightError,
)

class TestErrorHierarchy:
    """Tests for custom exception hierarchy."""

    def test_all_errors_inherit_from_base(self):
        """All custom errors inherit from CodepictureError."""
        for error_class in [ConfigError, ThemeError, RenderError, HighlightError]:
            assert issubclass(error_class, CodepictureError)

    def test_config_error_field_attribute(self):
        """ConfigError stores optional field name."""
        error = ConfigError("Invalid value", field="font_size")
        assert error.field == "font_size"
        assert "Invalid value" in str(error)

    def test_can_catch_all_codepicture_errors(self):
        """CodepictureError catch-all works."""
        with pytest.raises(CodepictureError):
            raise ConfigError("test")
```

### Anti-Patterns to Avoid

- **Global test state:** Never use module-level variables that persist between tests. pytest-randomly will expose this.
- **Direct file system access:** Always use `tmp_path` fixture for temporary files.
- **Testing Pydantic internals:** Test validation behavior, not Pydantic's implementation details.
- **Importing from conftest.py:** Fixtures are discovered automatically; never import conftest.

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Temp directories | `tempfile.mkdtemp()` | pytest `tmp_path` | Automatic cleanup, unique per test |
| Coverage tracking | Manual file tracking | pytest-cov | Proper branch coverage, exclusion patterns |
| Test randomization | Custom shuffle | pytest-randomly | Handles seed printing, reproduceability |
| Pydantic error parsing | String parsing | `ValidationError.errors()` | Structured error access |
| CI caching | Manual cache logic | setup-uv action cache | Built-in, maintained by Astral |

**Key insight:** pytest's built-in fixtures (`tmp_path`, `capsys`, `monkeypatch`) cover most testing needs. Adding custom fixtures should be rare and focused on project-specific setup.

## Common Pitfalls

### Pitfall 1: Forgetting to Register Custom Markers

**What goes wrong:** Running `pytest -m slow` shows warnings about unknown markers
**Why it happens:** Custom markers must be registered in pyproject.toml
**How to avoid:** Add `markers` list to `[tool.pytest.ini_options]`:
```toml
[tool.pytest.ini_options]
markers = [
    "slow: marks tests as slow (deselect with '-m \"not slow\"')",
]
```
**Warning signs:** "PytestUnknownMarkWarning" in test output

### Pitfall 2: FrozenInstanceError vs AttributeError

**What goes wrong:** `pytest.raises(FrozenInstanceError)` doesn't work
**Why it happens:** `FrozenInstanceError` is a subclass of `AttributeError`, but not always importable the same way
**How to avoid:** Use `pytest.raises(AttributeError)` - more portable
**Warning signs:** Import errors for `FrozenInstanceError`

### Pitfall 3: Coverage Not Excluding TYPE_CHECKING Blocks

**What goes wrong:** Coverage reports show uncovered lines in `if TYPE_CHECKING:` blocks
**Why it happens:** Using coverage.py < 7.10.0 or misconfigured exclusions
**How to avoid:** Use coverage >= 7.10.0 (automatic `TYPE_CHECKING` exclusion) or add explicit exclusion:
```toml
[tool.coverage.report]
exclude_also = [
    "if TYPE_CHECKING:",
]
```
**Warning signs:** Coverage showing misses on type-only import lines

### Pitfall 4: Test Order Dependencies

**What goes wrong:** Tests pass individually but fail when run together (or vice versa)
**Why it happens:** Shared mutable state between tests
**How to avoid:** pytest-randomly catches this. Use fixtures with proper scope.
**Warning signs:** Different results with `--randomly-seed=X` vs default

### Pitfall 5: conftest.py Import Issues

**What goes wrong:** `ImportError: cannot import name 'fixture' from 'conftest'`
**Why it happens:** Tests try to import from conftest.py directly
**How to avoid:** Never import from conftest.py - pytest discovers fixtures automatically
**Warning signs:** Import statements referencing conftest

### Pitfall 6: tmp_path Scope Mismatch

**What goes wrong:** Using `tmp_path` in a session-scoped fixture fails
**Why it happens:** `tmp_path` is function-scoped by default
**How to avoid:** Use `tmp_path_factory` for session/module-scoped fixtures
**Warning signs:** ScopeMismatch error from pytest

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| `tmpdir` fixture | `tmp_path` fixture | pytest 3.9+ | Returns pathlib.Path instead of py.path |
| `pytest.ini` | `pyproject.toml` | pytest 6.0+ (native TOML: 9.0+) | Consolidates all config in one file |
| Manual TYPE_CHECKING exclusion | Automatic | coverage.py 7.10.0 | No more pragma comments needed |
| pip install | uv sync | 2024+ | Much faster, lockfile-based |
| setup-python for uv projects | setup-uv | 2024+ | Native uv support in CI |

**Deprecated/outdated:**
- `tmpdir` / `tmpdir_factory`: Use `tmp_path` / `tmp_path_factory` instead (pathlib-based)
- `.coveragerc` file: Use `[tool.coverage.*]` in pyproject.toml
- `pytest.ini`: Use `[tool.pytest.ini_options]` in pyproject.toml (unless specific pytest.ini features needed)
